# 博客写作模板（每次写文章前发给 AI 用）

## 🤖 用法一：发给 AI 让它帮你写整篇文章

复制下面整段（含分割线之间的所有内容），粘贴到 Claude / ChatGPT / 文心 / 豆包，最后加上你的话题：

---

```
你是我的博客写作助手。我是西安石油大学机械工程的大四本科生，正在写毕设（电缆管道巡检机器人车体结构设计），平时也喜欢记录学习和生活杂记。

请帮我把下面这个话题写成一篇博客文章，要求：

【格式要求】
1. 输出必须是符合 Hexo 的 Markdown，开头必须包含 front-matter，格式如下：
   ---
   title: 文章标题
   date: 2026-05-26 10:00:00
   categories: [分类]
   tags: [标签1, 标签2]
   ---
2. 标题要朴素具体，不要标题党，不要"震惊体"。

- 第一人称口语化，像跟朋友聊天，不要"在当今社会""综上所述"那种官腔。
- 该有的小标题用 ## 二级，不要用一级 #（一级留给 Hexo 自己）。
- 关键句可以加粗 **这样**。
- 引用别人的话或自己的吐槽用 > 引用块。
- 代码块要标语言，比如 ```python。
- 文末用一两句自然收尾，不要"总结起来""综上所述"。

【长度】
- 杂记类：300-600 字
- 学习笔记类：600-1200 字
- 毕设进度类：400-800 字 + 关键截图占位

【话题】
（在这里写你想写什么，越具体越好。例：今天调 SolidWorks 装配体老是干涉，最后发现是螺栓孔尺寸标错了，想写一篇吐槽 + 教训。）

```

## 🤖 用法二：让 AI 润色你已经写好的草稿

```
帮我润色下面这篇博客草稿。要求：
1. 保留我的原意和口语风格，不要改成书面语。
2. 修语病、错别字、不通顺的句子。
3. 加合适的小标题（## 二级）让结构清晰。
4. 在开头补上 Hexo 的 front-matter（title、date 用今天、categories、tags）。
5. 不要加"总结"段落。

我的草稿：
（粘贴你的草稿）
```

## 🤖 用法三：让 AI 把零散笔记整理成文章

```
我有一些零散的笔记/想法，请帮我整理成一篇 Hexo 博客文章。
要求：
1. 开头加 Hexo 的 front-matter（categories 选"杂记"，tags 你自己想 2-3 个）。
2. 重新组织顺序，相关的合并到一起，加 ## 二级小标题。
3. 我的原话尽量保留，只补连接词让它读起来顺。
4. 文末加一句自然的收尾。

我的笔记（顺序乱的）：
（粘贴你的笔记）
```

# 💡 使用小贴士

### 关于 front-matter 的小知识

| 字段 | 必填？ | 说明 |
|---|---|---|
| `title` | ✅ | 文章标题，不要带特殊符号 `: " \ /` |
| `date` | ✅ | 格式 `YYYY-MM-DD HH:MM:SS`，决定首页排序 |
| `categories` | 推荐 | 一篇文章只放一个最贴切的分类 |
| `tags` | 推荐 | 2-4 个具体标签，方便以后检索 |
| `sticky` | 可选 | 写数字（如 100）会置顶到首页 |

### 写完之后怎么发

1. 把生成好的内容**保存为 `.md` 文件**（文件名随意）
2. 双击 `D:\BoKe\打开博客后台.bat`
3. 登录后**直接拖文件到网页**（或点"导入 Markdown 文件"）
4. 在编辑器里再看一遍、改一改
5. 点 **"发布上线"**

### 几个小习惯帮你写得更顺

- ✅ **先列大纲再写**：哪怕只有 3 个小标题
- ✅ **写完先丢一边**：吃个饭/睡一觉再回来读，错别字会自己冒出来
- ✅ **保留口语**：博客不是论文，"我觉得""我傻了""所以呢"都可以留着
- ❌ **别一次写太长**：超过 1500 字读者会划走，宁可拆成两篇
- ❌ **别用太多表情符号**：偶尔点缀就行，不然显得不正经
