# 写文章速查

## 最快流程

后台方式：

```powershell
cd /d "D:\BOKE plus"
npm run admin
```

然后打开：

```text
http://127.0.0.1:5050/
```

命令行方式：

```powershell
cd /d "D:\BOKE plus"
npm run new "文章标题"
npm run server
npm run publish
```

## 文章开头

每篇文章最上面都有一段信息，叫 Front Matter：

```markdown
---
title: 文章标题
date: 2026-05-25 21:00:00
categories:
  - 分类名
tags:
  - 标签1
  - 标签2
---
```

`categories` 是专栏/分类，通常 1 个就够。`tags` 是标签，可以多个。

## 常用 Markdown

```markdown
# 一级标题
## 二级标题
### 三级标题

这是一段正文。

**加粗**
*斜体*

- 列表项
- 列表项

![图片说明](/uploads/example.png)
```

## 文件链接

后台上传 PDF、DOCX 等文件后，会自动插入类似这样的链接：

```markdown
[文件名.pdf](/uploads/文件名.pdf)
```

敏感文件请使用后台的“安全上传”。安全上传会生成需要密码的打开链接，不会直接公开原文件。

## 开启评论

评论系统预留了 Giscus 配置。需要先在 GitHub 仓库 `WFCrush/boke-plus` 里开启 Discussions，然后访问：

```text
https://giscus.app/zh-CN
```

选择仓库后，把页面生成的 `repo-id`、`category`、`category-id` 填到 `_config.fluid.yml` 的 `giscus` 配置里，再把 `post.comments.enable` 改成 `true`。
