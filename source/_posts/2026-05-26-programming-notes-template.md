## 背景

刚开始写技术博客时，最容易卡住的不是技术本身，而是不知道一篇文章该怎么组织。对正在学习技术的人来说，一篇高质量笔记至少要回答三个问题：

- 我遇到了什么问题？
- 我如何理解和解决它？
- 下次遇到类似问题可以直接复用什么？

## 推荐结构

### 1. 问题背景

用几句话说明学习场景，例如课程作业、项目开发、环境配置或算法练习。

### 2. 核心知识

把最重要的概念、命令、代码片段整理出来。不要只贴代码，要写清楚为什么这样写。

```js
function sum(list) {
  return list.reduce((total, item) => total + item, 0);
}
```

### 3. 踩坑记录

这一部分最有价值。建议固定写成：

- 现象：报错或异常表现
- 原因：真正的问题点
- 解决：最终有效的方法

### 4. 总结

最后用 3-5 句话写清楚收获，方便以后快速回顾。

## 分类和标签

分类建议少而稳定，例如：

- 前端开发
- 后端开发
- 数据结构与算法
- 环境配置
- 学习方法

标签可以更具体，例如 `JavaScript`、`Node.js`、`Git`、`Hexo`、`算法题`。

## 总结

写技术笔记不需要追求一开始就完美。先把问题记录清楚，再不断补充代码、截图和复盘，博客就会慢慢变成自己的技术成长档案。
